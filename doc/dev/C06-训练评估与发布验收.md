# C06 - 训练评估与发布验收

## 1. 周期目标

补全训练与评估闭环，完成跨平台验收并形成可发布版本。

## 2. 输入与边界

输入：
- 需求文档 §2.1 F6、§3.2、§3.6
- 技术文档 §13、§14、§18

本周期包含：
- 离线训练管线
- CLER 对齐验证
- Windows 兼容性验证
- 发布文档整理

## 3. 任务清单

1. 训练管线
- 落地 `train.py/data.py/transforms.py`（复用原项目核心策略）。
- 支持 YAML 配置驱动训练，不依赖 Hydra。
- 输出 checkpoint 供实时推理加载。

2. 评估与对齐
- 集成 `cler.py` 评估链路。
- 对齐验证：
- 流式推理 CLER 与整段推理 CLER 差异 < 0.01。
- RT-Gesture 与原项目基准可对比。

3. 跨平台验证
- Linux 全流程回归。
- Windows + CUDA 环境启动、推理、GUI、关闭流程验证。

4. 发布准备
- 更新 README：安装、配置、启动、测试、常见故障。
- 固化 `requirements/environment` 与版本说明。
- 形成最终验收记录与后续路线图（真实硬件接入）。

## 4. 交付物

1. 训练脚本与评估脚本可执行。
2. 新 checkpoint 可直接用于 C03/C05 推理链路。
3. CLER 对齐报告与跨平台验证报告。
4. 发布候选版文档包。

## 5. 验收标准

1. 训练流程可在 full 数据集跑通并产出有效 checkpoint。
2. CLER 对齐满足预设阈值（差异 < 0.01）。
3. Linux 与 Windows 的关键流程均通过验收用例。
4. 文档可支持新成员从零部署到运行。

## 6. 风险与缓解

风险：
- Windows 驱动/CUDA/PyQt 组合差异导致兼容问题。

缓解：
- 形成最小支持版本矩阵，先锁定可用组合再扩展。

# C02 - 流式模型与事件检测

## 1. 周期目标

完成实时识别的算法核心：`forward_streaming`、checkpoint 加载、事件检测三层过滤。

## 2. 输入与边界

输入：
- 需求文档 §2.1 F2/F3、§3.4
- 技术文档 §4、§5、§6、§14

本周期包含：
- 流式推理状态设计（conv history + LSTM state）
- Lightning checkpoint 权重映射加载
- 事件检测（休息态抑制、拒识、去抖）
- 算法一致性单测

本周期不包含：
- 跨进程通信与真实实时数据接入
- GUI 可视化

## 3. 任务清单

1. `networks.py`
- 复用原模型结构并新增 `forward_streaming()`。
- 明确 `left_context=20`、`stride=10`、输出 200Hz。
- 支持 `T_out=0` 的边界输入。

2. `checkpoint_utils.py`
- 加载 Lightning checkpoint（`weights_only=False`）。
- 剥离 `network.` 前缀后 `strict=True` 加载。
- 自动设备选择：CUDA 优先，CPU 回退。

3. `event_detector.py`
- 状态字段：`prev_probs`、`last_event_time`。
- 规则顺序：静息抑制 -> 上穿检测 -> 拒识 -> 去抖。
- 提供 `reset()` 用于中断恢复。

4. 单元测试
- `forward()` 与流式拼接结果 `allclose(atol=1e-6)`。
- 多 chunk_size 一致性（10/20/40/100）。
- 阈值、拒识、去抖、reset 行为测试。

## 4. 交付物

1. `networks.py`、`checkpoint_utils.py`、`event_detector.py` 可独立运行。
2. 算法一致性测试通过并输出记录。
3. 事件检测输出格式统一为 `(gesture, timestamp, confidence)`。

## 5. 验收标准

1. 流式推理与整段推理在 `eval()` 下一致。
2. checkpoint 20 个关键 tensor 全量匹配加载成功。
3. 事件检测满足默认阈值策略：
- `threshold=0.35`
- `rejection_threshold=0.6`
- `rest_threshold=0.15`
- `debounce=0.05s`

## 6. 风险与缓解

风险：
- 流式时间对齐与整段推理错位，导致 CLER 偏差。

缓解：
- 在本周期先完成数值一致性，时间轴对齐在 C03/C04 联调验证。

# C05 - GUI监控系统

## 1. 周期目标

交付可用于演示和调参的桌面监控端，实现实时波形、概率、事件与系统状态可视化。

## 2. 输入与边界

输入：
- 需求文档 §2.1 F5
- 技术文档 §12

本周期包含：
- 主窗口与布局
- ZMQ 读线程 + 主线程渲染
- 配置面板、控制按钮、状态栏

本周期不包含：
- 模型训练逻辑
- 新的推理算法改造

## 3. 任务清单

1. GUI 架构
- `main_window.py` 组装 `ConfigPanel/EmgPlotWidget/GestureDisplay/StatusBar`。
- `QThread` 持有 ZMQ SUB，主线程仅做渲染更新。

2. 实时可视化
- 16 通道 EMG 波形滚动显示。
- 9 类手势概率条形图或热力图。
- 事件日志列表（时间、类别、置信度）。

3. 操作与配置
- 启动/停止、暂停/恢复显示。
- 参数选择：数据文件、checkpoint、device、阈值。
- 状态显示：FPS、延迟、设备状态、连接状态。

4. 性能优化
- `setData` 增量更新，避免重建曲线。
- 5 秒滚动窗口缓存，必要时降采样显示。

5. GUI 测试
- 基础交互测试（控件、状态切换）。
- 长时渲染稳定性测试（>=10 分钟）。

## 4. 交付物

1. 可启动的 GUI 监控应用。
2. 与实时推理进程联动的配置/控制能力。
3. GUI 操作说明文档。

## 5. 验收标准

1. EMG 波形可稳定刷新，刷新率 `>= 30 FPS`。
2. 概率与事件列表可实时跟随推理输出更新。
3. 停止/关闭操作不会导致 GUI 卡死或后台进程残留。

## 6. 风险与缓解

风险：
- 主线程阻塞导致渲染掉帧。

缓解：
- ZMQ 接收严格放在独立线程，UI 更新通过信号槽异步派发。

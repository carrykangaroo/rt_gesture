# RT-Gesture 开发周期总览

> 依据文档：
> - `doc/实时离散手势识别系统需求规格说明书.md`（v0.2）
> - `doc/技术实现文档.md`（v1.0）
> - 生成日期：2026-03-07

## 1. 划分原则

1. 先打基础再闭环：先完成环境、骨架、模型流式能力，再做端到端联调。
2. 以验收为单位拆周期：每个周期结束都必须有可验证的可交付物。
3. 风险前置：把 LSTM 状态一致性、数据中断恢复、延迟预算验证提前。
4. 可并行处并行：GUI 与训练管线放在核心 MVP 稳定后并行推进。

## 2. 周期总表

| 周期 | 周期名称 | 目标 | 关键输出 | 建议工时 |
|---|---|---|---|---|
| C01 | 环境与工程骨架 | 建立可开发、可运行、可测试的基础工程 | 配置体系、常量、目录、依赖安装脚本 | 8h |
| C02 | 流式模型与事件检测 | 完成模型流式推理与后处理核心算法 | `networks/checkpoint/event_detector` + 单测 | 12h |
| C03 | 实时管道与MVP闭环 | 跑通数据模拟→推理→事件输出 | `zmq/data_sim/inference/main/logger` + 集成链路 | 14h |
| C04 | 稳定性与性能达标 | 满足中断恢复、优雅关闭、延迟预算 | 异常恢复、延迟报告、性能/关闭测试 | 10h |
| C05 | GUI监控系统 | 交付可操作的实时监控桌面端 | PyQt6 GUI、配置面板、状态监控 | 10h |
| C06 | 训练评估与发布验收 | 建立训练复现与跨平台验收能力 | 训练脚本、CLER 对齐、Windows 验证、发布文档 | 8h |

总计建议工时：约 62h（与技术文档估算 60.5h 基本一致）。

## 3. 周期依赖关系

`C01 -> C02 -> C03 -> C04`

`C05` 依赖 `C03`（基础管道稳定后启动）  
`C06` 依赖 `C02`（模型与评估基础就绪），建议在 `C04/C05` 期间并行推进

## 4. 统一验收口径（所有周期适用）

1. 代码质量：新增模块必须有对应单测或集成测试。
2. 配置可控：关键参数均可从 YAML 配置覆盖。
3. 可追踪性：关键行为有日志，关键结果可存档（JSONL/NPZ）。
4. 性能约束：最终系统满足端到端延迟 `<= 200ms`，GUI 刷新 `>= 30 FPS`。
5. 稳定性约束：数据中断触发 LSTM 重置 + warm-up，关闭过程无僵尸进程与端口残留。
